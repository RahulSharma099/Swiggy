# Repository Development Skill

## Purpose

Repositories encapsulate all database queries and Prisma interactions. They provide a clean API for services to access data without knowing about Prisma internals. All repositories are built using functional composition.

---

## Core Repository Factory Pattern

Repositories are factory functions that return objects of pure database query functions.

```typescript
// repositories/issue.repository.ts

/**
 * Creates an issue repository with all database access functions.
 *
 * @param prisma - Prisma client instance
 * @param logger - Winston logger instance
 * @returns Object containing all issue query functions
 */
export const createIssueRepository = (prisma: PrismaClient, logger: Logger) => {
  return {
    /**
     * Find issue by ID within workspace (authorization check)
     *
     * @param issueId - Issue ID to find
     * @param workspaceId - Workspace context for authorization
     * @returns Issue with related data, or null if not found or not authorized
     */
    async findByIdInWorkspace(
      issueId: string,
      workspaceId: string,
    ): Promise<Issue | null> {
      return prisma.issue.findFirst({
        where: {
          id: issueId,
          project: { workspaceId }, // Workspace authorization
        },
        include: {
          project: true,
          assignee: true,
          watchers: true,
          comments: { take: 10, orderBy: { createdAt: "desc" } },
        },
      });
    },

    /**
     * Find issues by project with optional filtering
     *
     * @param projectId - Project ID
     * @param options - Query options (filters, pagination, includes)
     * @returns Array of issues
     */
    async findByProject(
      projectId: string,
      options?: {
        status?: string[];
        priority?: string[];
        offset?: number;
        limit?: number;
      },
    ): Promise<Issue[]> {
      return prisma.issue.findMany({
        where: {
          projectId,
          status: options?.status ? { in: options.status } : undefined,
          priority: options?.priority ? { in: options.priority } : undefined,
        },
        skip: options?.offset || 0,
        take: options?.limit || 20,
        orderBy: { createdAt: "desc" },
      });
    },

    /**
     * Create issue and related audit log in transaction
     *
     * @param data - Issue creation data
     * @param projectId - Project ID
     * @param userId - User creating the issue
     * @returns Created issue
     */
    async create(
      data: Omit<IssueCreateInput, "projectId">,
      projectId: string,
      userId: string,
    ): Promise<Issue> {
      return prisma.$transaction(async (tx) => {
        const issue = await tx.issue.create({
          data: {
            ...data,
            projectId,
            creatorId: userId,
            version: 1,
          },
          include: { project: true, assignee: true },
        });

        // Activity log created in transaction
        await tx.activityLog.create({
          data: {
            issueId: issue.id,
            projectId,
            actionType: "created",
            changedFields: { title: data.title },
            actorId: userId,
          },
        });

        return issue;
      });
    },

    /**
     * Update issue with optimistic locking check
     *
     * @param issueId - Issue ID to update
     * @param expectedVersion - Current version (for optimistic locking)
     * @param data - Fields to update
     * @returns Updated issue, or null if version mismatch
     */
    async updateWithVersionCheck(
      issueId: string,
      expectedVersion: number,
      data: Partial<Issue>,
    ): Promise<Issue | null> {
      const updated = await prisma.issue
        .update({
          where: {
            id: issueId,
            version: expectedVersion, // Optimistic lock check
          },
          data: {
            ...data,
            version: expectedVersion + 1, // Increment version
          },
          include: { project: true, assignee: true },
        })
        .catch(() => null); // Catch unique constraint violation

      if (!updated) {
        logger.warn("Version mismatch on issue update", {
          issueId,
          expectedVersion,
        });
        return null;
      }

      return updated;
    },

    /**
     * Get issue hierarchy (parent + epic for given issue)
     *
     * @param issueId - Issue ID
     * @returns Issue with hierarchy info
     */
    async getHierarchy(issueId: string): Promise<IssueHierarchy | null> {
      const issue = await prisma.issue.findUnique({
        where: { id: issueId },
        include: {
          parent: {
            include: { parent: true },
          },
          epic: true,
          children: true,
        },
      });

      if (!issue) return null;

      return {
        issue,
        parent: issue.parent,
        epic: issue.epic,
        hasChildren: issue.children.length > 0,
      };
    },

    /**
     * Get issues by custom field value
     * Supports dynamic custom fields stored in JSONB
     *
     * @param customFieldId - Custom field ID
     * @param value - Value to match
     * @param projectId - Project (optional, for scoping)
     * @returns Matching issues
     */
    async findByCustomField(
      customFieldId: string,
      value: unknown,
      projectId?: string,
    ): Promise<Issue[]> {
      return prisma.issue.findMany({
        where: {
          ...(projectId && { projectId }),
          customFields: {
            path: [customFieldId],
            equals: value,
          },
        },
      });
    },

    /**
     * Full-text search issues by title and description
     *
     * @param query - Search term
     * @param projectId - Project scope
     * @param limit - Max results
     * @returns Matching issues with relevance score
     */
    async searchFullText(
      query: string,
      projectId: string,
      limit: number = 50,
    ): Promise<(Issue & { relevance: number })[]> {
      const results = await prisma.$queryRaw<(Issue & { relevance: number })[]>`
        SELECT 
          i.*,
          ts_rank(to_tsvector('english', i.title || '' || ' ' || i.description || ''), 
                  plainto_tsquery('english', ${query})) as relevance
        FROM "Issue" i
        WHERE i."projectId" = ${projectId}
          AND to_tsvector('english', i.title || '' || ' ' || i.description || '') 
              @@ plainto_tsquery('english', ${query})
        ORDER BY relevance DESC
        LIMIT ${limit}
      `;

      return results;
    },

    /**
     * Batch update multiple issues in transaction
     *
     * @param issueIds - Issue IDs to update
     * @param data - Fields to update for all
     * @returns Count of updated issues
     */
    async batchUpdate(
      issueIds: string[],
      data: Partial<Issue>,
    ): Promise<number> {
      if (issueIds.length === 0) return 0;

      const result = await prisma.issue.updateMany({
        where: { id: { in: issueIds } },
        data,
      });

      logger.info("Batch update completed", {
        count: result.count,
        issueIds: issueIds.length,
      });
      return result.count;
    },

    /**
     * Soft delete issue (mark as deleted)
     *
     * @param issueId - Issue to delete
     * @param userId - User performing delete (for audit)
     * @returns Updated issue with deletedAt set
     */
    async softDelete(issueId: string, userId: string): Promise<Issue> {
      return prisma.$transaction(async (tx) => {
        const issue = await tx.issue.update({
          where: { id: issueId },
          data: { deletedAt: new Date() },
        });

        await tx.activityLog.create({
          data: {
            issueId,
            projectId: issue.projectId,
            actionType: "deleted",
            actorId: userId,
          },
        });

        return issue;
      });
    },
  };
};

export type IssueRepository = ReturnType<typeof createIssueRepository>;
```

## Query Patterns

### 1. Efficient Loading (Prevents N+1)

```typescript
// ❌ BAD: N+1 queries
async function getBadIssues(projectId: string) {
  const issues = await prisma.issue.findMany({ where: { projectId } });
  const withAssignees = await Promise.all(
    issues.map((i) => prisma.user.findUnique({ where: { id: i.assigneeId } })),
  );
  return { issues, assignees: withAssignees };
}

// ✅ GOOD: Single query with include
async function getGoodIssues(projectId: string) {
  return prisma.issue.findMany({
    where: { projectId },
    include: {
      assignee: true,
      project: true,
      comments: { take: 5, orderBy: { createdAt: "desc" } },
    },
  });
}
```

### 2. Pagination

```typescript
// Offset-based (simple)
async function listWithOffset(
  workspaceId: string,
  offset: number = 0,
  limit: number = 20,
) {
  const [items, total] = await prisma.$transaction([
    prisma.issue.findMany({
      where: { project: { workspaceId } },
      skip: offset,
      take: limit,
      orderBy: { createdAt: "desc" },
    }),
    prisma.issue.count({
      where: { project: { workspaceId } },
    }),
  ]);

  return { items, total, offset, limit, hasMore: offset + limit < total };
}

// Cursor-based (efficient for large datasets)
async function listWithCursor(
  workspaceId: string,
  cursor?: string,
  limit: number = 20,
) {
  const items = await prisma.issue.findMany({
    where: { project: { workspaceId } },
    take: limit + 1, // Fetch one extra to determine hasMore
    cursor: cursor ? { id: cursor } : undefined,
    skip: cursor ? 1 : 0, // Skip the cursor itself
    orderBy: { id: "asc" },
  });

  const hasMore = items.length > limit;
  const nextCursor = hasMore ? items[limit - 1].id : null;

  return {
    items: items.slice(0, limit),
    nextCursor,
    hasMore,
  };
}
```

### 3. Transactions for Multi-Step Operations

```typescript
async function moveIssuesToSprintAndNotify(
  issueIds: string[],
  sprintId: string,
  userId: string,
) {
  return prisma.$transaction(async (tx) => {
    // Step 1: Update issues
    const updated = await tx.issue.updateMany({
      where: { id: { in: issueIds } },
      data: { sprintId, updatedAt: new Date() },
    });

    // Step 2: Create notifications
    const notifications = await tx.notification.createMany({
      data: issueIds.map((issueId) => ({
        userId,
        type: "ISSUE_MOVED",
        relatedIssueId: issueId,
      })),
    });

    // Both succeed or both fail
    return { updated: updated.count, notifications: notifications.count };
  });
}
```

### 4. Workspace Authorization in Queries

```typescript
// EVERY query must filter by workspace
async function getIssue(issueId: string, workspaceId: string) {
  return prisma.issue.findFirst({
    where: {
      id: issueId,
      // Cross-workspace queries IMPOSSIBLE
      project: { workspaceId },
    },
  });
}

// ❌ NEVER do this (allows cross-workspace access!)
async function getBadQuery(issueId: string) {
  return prisma.issue.findUnique({ where: { id: issueId } });
}
```

### 5. Complex Filtering with JSONB

```typescript
async function findByCustomFieldCondition(
  projectId: string,
  fieldId: string,
  condition: { operator: "eq" | "gt" | "lt" | "contains"; value: unknown },
) {
  const conditionMap = {
    eq: `"conditions"->>'${fieldId}' = '${condition.value}'`,
    gt: `("conditions"->>'${fieldId}')::numeric > ${condition.value}`,
    lt: `("conditions"->>'${fieldId}')::numeric < ${condition.value}`,
    contains: `"conditions"->>'${fieldId}' LIKE '%${condition.value}%'`,
  };

  return prisma.$queryRaw`
    SELECT * FROM "Issue"
    WHERE "projectId" = ${projectId}
      AND ${prisma.raw(conditionMap[condition.operator])}
  `;
}
```

### 6. Aggregations

```typescript
async function getProjectStats(projectId: string) {
  const stats = await prisma.issue.groupBy({
    by: ["status"],
    where: { projectId },
    _count: true,
  });

  return stats.reduce((acc, s) => ({ ...acc, [s.status]: s._count }), {});
}
```

## Creating Repository Functions

### Checklist

- [ ] JSDoc with @param, @returns, @throws
- [ ] Workspace authorization check (`.where: { project: { workspaceId } }`)
- [ ] Efficient loading (`.include` for relations, not N+1)
- [ ] Pagination parameters (offset/limit or cursor)
- [ ] Transactions for multi-step operations
- [ ] Logging for mutations and errors
- [ ] Optimistic locking version checks
- [ ] Soft delete support (check `deletedAt` in queries)
- [ ] Type safety (use Prisma-generated types)
- [ ] No raw Prisma in services (all through repo)

---

## Repository Function Categories

### Read Functions

- `findById(id, workspaceId)` - Get single item
- `findByIdInWorkspace(...)` - With auth
- `findMany(where, options)` - Query with filters
- `listPaginated(...)` - With offset pagination
- `listCursor(...)` - With cursor pagination
- `count(where)` - Count matching items

### Write Functions

- `create(data, userId)` - Create new item
- `update(id, data, userId)` - Update one
- `updateWithVersionCheck(...)` - Optimistic lock
- `batchUpdate(ids, data)` - Update many
- `delete(id)` - Hard delete (rare)
- `softDelete(id, userId)` - Mark deleted

### Search Functions

- `searchFullText(query, ...)` - Full-text search
- `findByCustomField(...)` - Dynamic fields
- `searchComplex(filters, ...)` - Complex queries

### Aggregation Functions

- `count(where)` - Count items
- `getStats(...)` - Aggregated data
- `findRelated(...)` - Related items

## Best Practices

✅ **DO**:

- Include related data eagerly with `.include`
- Use transactions for multi-step operations
- Check workspace in every query
- Version check before updates (optimistic locking)
- Log all mutations
- Paginate large result sets
- Use JSONB for extensibility

❌ **DON'T**:

- Call Prisma directly from services
- Forget workspace authorization
- N+1 queries (use include, not loops)
- Mix business logic with data access
- Return raw Prisma objects to services (transform first)
- Skip version checks on updates
- Log sensitive data
