# Workflow Engine Development Skill

## Purpose

The workflow engine manages the valid transitions between issue statuses and executes associated actions. It's a state machine that enforces business rules, executes transitions, and emits events. Built using functional composition.

---

## Workflow Model

Issues have a **status** that transitions through a configured workflow. Each workflow is project-specific and can be customized.

```
┌─────────┐      ┌─────────┐      ┌────────┐      ┌─────┐
│ Backlog │─────→│ Todo    │─────→│ In Prog│─────→│ Done│
└─────────┘      └─────────┘      └────────┘      └─────┘
                       ↑                ↓
                       └────────────────┘
                     (Reject to Backlog)
```

---

## Workflow Configuration

Workflows are stored in the database and can be customized per project:

```typescript
// Database schema
export interface WorkflowDefinition {
  id: string;
  projectId: string;
  name: string; // e.g., "Agile Workflow"
  statuses: Status[];
  transitions: Transition[];
}

export interface Status {
  id: string;
  name: string; // backlog, todo, in_progress, done, etc.
  isInitial: boolean; // Starting status for new issues
  isFinal: boolean; // Terminal status
}

export interface Transition {
  id: string;
  from: string; // Status ID
  to: string; // Status ID
  conditions: Condition[]; // Must ALL be true
  actions: Action[]; // Executed after transition
  requiredRole?: "viewer" | "developer" | "maintainer"; // Optional permission check
}

export interface Condition {
  type: "has_assignee" | "has_reviewers" | "custom_field" | "sprint_active";
  config: Record<string, unknown>;
}

export interface Action {
  type: "notify_watchers" | "update_field" | "create_activity" | "webhook";
  config: Record<string, unknown>;
}
```

## Workflow Engine Factory

```typescript
// services/workflow.engine.ts

/**
 * Creates a workflow engine for managing issue status transitions
 *
 * @param deps - Dependencies (repositories, logger, event publisher)
 * @returns Workflow engine with transition methods
 */
export const createWorkflowEngine = (deps: {
  workflowRepo: WorkflowRepository;
  issueRepo: IssueRepository;
  activityRepo: ActivityRepository;
  logger: Logger;
  eventPublisher: EventPublisher;
}) => {
  const { workflowRepo, issueRepo, activityRepo, logger, eventPublisher } =
    deps;

  return {
    /**
     * Get workflow configuration for a project
     *
     * @param projectId - Project ID
     * @returns Workflow definition, or null if not found
     */
    async getWorkflow(projectId: string): Promise<WorkflowDefinition | null> {
      return workflowRepo.findByProjectId(projectId);
    },

    /**
     * Check if a status transition is allowed
     *
     * @param projectId - Project ID
     * @param fromStatus - Current status
     * @param toStatus - Desired status
     * @param issue - Current issue (for condition evaluation)
     * @param userId - User attempting transition (for permission check)
     * @returns True if transition is allowed
     */
    async canTransition(
      projectId: string,
      fromStatus: string,
      toStatus: string,
      issue: Issue,
      userId: string,
    ): Promise<boolean> {
      // Get workflow
      const workflow = await this.getWorkflow(projectId);
      if (!workflow) {
        logger.warn("Workflow not found", { projectId });
        return false;
      }

      // Find transition
      const transition = workflow.transitions.find(
        (t) => t.from === fromStatus && t.to === toStatus,
      );

      if (!transition) {
        logger.debug("Transition not defined", { fromStatus, toStatus });
        return false;
      }

      // Check permission
      if (transition.requiredRole) {
        const hasPermission = await checkUserPermission(
          userId,
          projectId,
          transition.requiredRole,
        );
        if (!hasPermission) {
          logger.warn("User lacks permission for transition", {
            userId,
            transition: `${fromStatus} → ${toStatus}`,
          });
          return false;
        }
      }

      // Evaluate all conditions
      for (const condition of transition.conditions) {
        const met = await evaluateCondition(condition, issue);
        if (!met) {
          logger.debug("Condition not met", { condition: condition.type });
          return false;
        }
      }

      return true;
    },

    /**
     * Execute a status transition
     *
     * @param projectId - Project ID
     * @param issueId - Issue to transition
     * @param toStatus - Target status
     * @param userId - User performing transition
     * @returns Updated issue
     * @throws UnprocessableEntityError if transition not allowed
     */
    async transitionIssue(
      projectId: string,
      issueId: string,
      toStatus: string,
      userId: string,
    ): Promise<Issue> {
      // Get issue
      const issue = await issueRepo.findByIdInWorkspace(issueId, projectId);
      if (!issue) {
        throw new NotFoundError("Issue", issueId);
      }

      const fromStatus = issue.status;

      // Check if transition is allowed
      const allowed = await this.canTransition(
        projectId,
        fromStatus,
        toStatus,
        issue,
        userId,
      );

      if (!allowed) {
        throw new UnprocessableEntityError(
          `Cannot transition from ${fromStatus} to ${toStatus}`,
          "INVALID_TRANSITION",
        );
      }

      // Get workflow to find transition
      const workflow = await this.getWorkflow(projectId);
      const transition = workflow!.transitions.find(
        (t) => t.from === fromStatus && t.to === toStatus,
      );

      // Execute transition in transaction
      const updated = await issueRepo.updateWithVersionCheck(
        issueId,
        issue.version,
        { status: toStatus },
      );

      if (!updated) {
        throw new ConflictError(
          "Issue was modified",
          "CONFLICT_VERSION_MISMATCH",
        );
      }

      // Execute transition actions
      if (transition?.actions) {
        await this.executeActions(transition.actions, updated, userId);
      }

      // Log activity
      await activityRepo.log({
        issueId,
        projectId,
        actionType: "transitioned",
        changedFields: { status: { from: fromStatus, to: toStatus } },
        actorId: userId,
      });

      // Emit event
      await eventPublisher.emit("issue_status_changed", {
        issueId,
        projectId,
        from: fromStatus,
        to: toStatus,
        transitionedBy: userId,
        timestamp: new Date().toISOString(),
      });

      logger.info("Issue transitioned", {
        issueId,
        fromStatus,
        toStatus,
        userId,
      });

      return updated;
    },

    /**
     * Execute post-transition actions
     *
     * @param actions - Actions to execute
     * @param issue - Issue that was transitioned
     * @param userId - User who initiated transition
     */
    async executeActions(
      actions: Action[],
      issue: Issue,
      userId: string,
    ): Promise<void> {
      for (const action of actions) {
        try {
          await executeAction(action, issue, userId);
        } catch (error) {
          logger.error("Action execution failed", {
            action: action.type,
            issueId: issue.id,
            error,
          });
          // Don't throw; log but continue with other actions
        }
      }
    },

    /**
     * Get available transitions from current status
     *
     * @param projectId - Project ID
     * @param currentStatus - Current status
     * @returns Array of possible target statuses
     */
    async getAvailableTransitions(
      projectId: string,
      currentStatus: string,
    ): Promise<string[]> {
      const workflow = await this.getWorkflow(projectId);
      if (!workflow) return [];

      return workflow.transitions
        .filter((t) => t.from === currentStatus)
        .map((t) => t.to);
    },
  };
};

export type WorkflowEngine = ReturnType<typeof createWorkflowEngine>;
```

## Condition Evaluation

```typescript
// Evaluate workflow conditions

async function evaluateCondition(
  condition: Condition,
  issue: Issue,
): Promise<boolean> {
  switch (condition.type) {
    case "has_assignee":
      return !!issue.assigneeId;

    case "has_reviewers":
      const reviewerCount = (condition.config.minReviewers as number) || 1;
      return (issue.reviewerIds?.length || 0) >= reviewerCount;

    case "custom_field":
      const fieldId = condition.config.fieldId as string;
      const expectedValue = condition.config.value;
      const fieldValue = issue.customFields?.[fieldId];
      return fieldValue === expectedValue;

    case "sprint_active":
      return !!issue.sprintId && issue.sprint?.status === "active";

    default:
      return true;
  }
}
```

## Action Execution

```typescript
// Execute post-transition actions

async function executeAction(
  action: Action,
  issue: Issue,
  userId: string,
): Promise<void> {
  switch (action.type) {
    case "notify_watchers":
      await sendNotificationsToWatchers(issue, action.config);
      break;

    case "update_field":
      await updateIssueField(issue.id, action.config);
      break;

    case "create_activity":
      await createCustomActivity(issue, action.config, userId);
      break;

    case "webhook":
      await callWebhook(issue, action.config);
      break;

    default:
      throw new Error(`Unknown action type: ${action.type}`);
  }
}

// Specific action implementations

async function sendNotificationsToWatchers(
  issue: Issue,
  config: { message?: string },
): Promise<void> {
  const watchers = await issueRepo.getWatchers(issue.id);

  await Promise.all(
    watchers.map((watcher) =>
      notificationService.send({
        userId: watcher.userId,
        type: "ISSUE_TRANSITIONED",
        title: `Issue transitioned`,
        body: config.message || `${issue.key} moved to ${issue.status}`,
        relatedIssueId: issue.id,
      }),
    ),
  );
}

async function updateIssueField(
  issueId: string,
  config: { field: string; value: unknown },
): Promise<void> {
  await issueRepo.update(issueId, {
    [config.field]: config.value,
  });
}

async function callWebhook(
  issue: Issue,
  config: { url: string; headers?: Record<string, string> },
): Promise<void> {
  try {
    await fetch(config.url, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        ...config.headers,
      },
      body: JSON.stringify({
        event: "issue_transition",
        issue,
        timestamp: new Date().toISOString(),
      }),
    });
  } catch (error) {
    logger.error("Webhook call failed", { url: config.url, error });
    throw error;
  }
}
```

## Integration with Issue Service

```typescript
// In IssueService, use workflow engine to validate transitions

export const createIssueService = (deps: ServiceDeps) => ({
  async transitionIssue(
    issueId: string,
    newStatus: string,
    expectedVersion: number,
    workspaceId: string,
    userId: string,
  ): Promise<Issue> {
    // Get issue
    const issue = await deps.repos.issue.findByIdInWorkspace(
      issueId,
      workspaceId,
    );
    if (!issue) {
      throw new NotFoundError("Issue", issueId);
    }

    // Use workflow engine to execute transition
    return deps.workflowEngine.transitionIssue(
      issue.projectId,
      issueId,
      newStatus,
      userId,
    );
  },

  async getIssueStatus(
    issueId: string,
    workspaceId: string,
  ): Promise<{ status: string; availableTransitions: string[] }> {
    const issue = await deps.repos.issue.findByIdInWorkspace(
      issueId,
      workspaceId,
    );
    if (!issue) {
      throw new NotFoundError("Issue", issueId);
    }

    const availableTransitions =
      await deps.workflowEngine.getAvailableTransitions(
        issue.projectId,
        issue.status,
      );

    return {
      status: issue.status,
      availableTransitions,
    };
  },
});
```

## Workflow Engine Checklist

- [ ] Workflow definition loaded per project
- [ ] Transitions checked before mutation
- [ ] All conditions evaluated before allowing transition
- [ ] Permission checks enforced
- [ ] Actions executed asynchronously (don't block response)
- [ ] Transition success logged
- [ ] Event emitted for real-time sync
- [ ] Activity log entry created
- [ ] Errors handled gracefully
- [ ] Default workflows provided or custom workflows supported
- [ ] Type-safe condition and action evaluation
- [ ] Tests cover valid and invalid transitions

---

## Key Principles

✅ **DO**:

- Define workflows per project
- Check all conditions before transition
- Execute actions asynchronously
- Emit events after successful transition
- Log all transitions
- Provide sensible defaults
- Support custom workflows
- Validate user permissions

❌ **DON'T**:

- Skip condition evaluation
- Allow invalid transitions
- Block response waiting for actions
- Hardcode statuses
- Mix workflow logic with service logic
- Silently fail actions
- Emit events before transition succeeds
